# 汉献帝之末路 · 产品设计

## 核心定位

**类型**：历史角色扮演沙盘模拟
**玩家**：扮演汉献帝刘协，在董卓乱政、诸侯割据的时代中艰难求存、力图中兴
**目标**：通过诏令、召对、用人、决策，延缓汉室灭亡乃至实现中兴

## 核心玩法循环

```
回合（月）= 召对 + 诏令 + 召幸 → LLM 模拟 → 结果落库 + 记忆生成 → 回合报告
```

1. **召对**：与大臣对话，了解局势、获取建议、观察态度
2. **拟旨**：将口谕转为正式诏书（LLM 润色）
3. **执行**：诏书通过 LLM 推演落地（数值变化 + 叙事生成）
4. **记忆**：系统提取本回合关键事件为记忆，影响后续召对
5. **报告**：生成月末奏章，呈献帝御览
6. **召幸**（v1.15.0+）：与后宫妃嫔对话、调教其技能/性格，衣带诏密谋可由此起
7. **候选情势判选**（v1.16.0+）：每月 LLM 软筛候选情势清单，避免与本月盘面冲突

## 派系系统

- 何进派（外戚）、董卓派（凉州军）、袁绍派（世家大族）
- 清流派（士人）、宦官派（内廷）
- 曹操派（寒门功臣）、刘备派（汉室宗亲）

## 特色机制

- **威权系统**：随都城迁移和重大事件剧烈波动
- **迁都机制**：洛阳→长安→许昌三都流转，每迁都有代价
- **历史锚点**：25个关键时间节点必须尊重（董卓进京/诸侯讨董/赤壁/禅让等）
- **董卓伏诛**：基于威权/军事/事件综合判定
- **献帝东归**：董卓死后的生存路线选择
- **藩镇动态**：各地节度使/太守自主决策

## 后宫系统（v1.15.0+ / Phase F v1.17.0）

**汉献帝特色**：后宫不只闺阁，更是密谋载体（衣带诏）。

- **6 位妃嫔**：伏寿（皇后·衣带诏主谋）/ 董贵（贵人·衣带诏）/ 曹贵人（曹操之女·政治婚姻）/ 李婉（嫔·虚构）/ 何莹（何进侄女·外戚血脉）/ 王美人（王允族女）
- **位份排序**：皇后 > 贵妃 > 妃 > 嫔 > 贵人 > 常在 > 答应
- **召幸对话**：按位份+性格+家族+派系定调（皇后端庄有锋芒、贵人谨言慎行）
- **调教工具** `cultivate_consort(skill, trait, emperor_words)`：学技能/改性格 → 永久性格写入 `consort_traits` 表
- **衣带诏线索**：家族相关+献帝主动问才可枕边进言
- **Web 后宫 Tab**（v1.17.0）：3 区布局——名册 / 召幸 / 调教 / 衣带诏线索
- **7 后端 API**：`/consorts` `/consorts/<id>` `/audience` `/records` `/cultivate` `/traits` `/consort_tab`

## 候选情势判选（v1.16.0+ / Phase E）

**双层软筛**：
1. 程序内 `gather_candidate_events`（trigger_window + trigger_gate）
2. LLM 因果判选（`event_selector` agent）
3. 退避：连续 hold 3 次强制 fire
4. 缓存：同盘面 24h 复用
5. 降级：LLM 失败时全量 fire

**汉献帝特殊判据**：
- active_issues ≥ 6 时少出（喘息窗口）
- active_issues = 0 时必出 1 条
- 董卓未死时王允相关 hold
- 曹操未控许时迎帝相关 hold
